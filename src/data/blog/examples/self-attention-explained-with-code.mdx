Self-attention is the core idea behind Transformers. If you've ever wondered how a model decides which words to focus on while reading a sentence, this is it.

## Why self-attention matters

In an RNN, each token is processed step by step. In self-attention, every token can directly “look at” every other token in the same sequence.

That gives us two huge advantages:

- Better long-range dependency handling
- Full parallelism during training

## The core idea in one sentence

For each token, compute a weighted average of all token representations, where weights come from token-to-token relevance scores.

## Query, Key, Value (Q, K, V)

Each input embedding `x_i` is linearly projected into three vectors:

- **Query**: what this token is looking for
- **Key**: what this token offers
- **Value**: the information this token carries

Mathematically:

$$
Q = XW_Q,\quad K = XW_K,\quad V = XW_V
$$

Then attention scores are:

$$
\mathrm{Attention}(Q, K, V) = \mathrm{softmax}(QK^T / \sqrt{d_k}) V
$$

## Minimal NumPy implementation

```python
import numpy as np

def softmax(x, axis=-1):
    x = x - np.max(x, axis=axis, keepdims=True)  # numerical stability
    exp_x = np.exp(x)
    return exp_x / np.sum(exp_x, axis=axis, keepdims=True)

def scaled_dot_product_attention(X, W_q, W_k, W_v, mask=None):
    """
    X:    (seq_len, d_model)
    W_q:  (d_model, d_k)
    W_k:  (d_model, d_k)
    W_v:  (d_model, d_v)
    mask: (seq_len, seq_len), optional. 1=keep, 0=block
    """
    Q = X @ W_q                         # (seq_len, d_k) # [!code highlight]
    K = X @ W_k                         # (seq_len, d_k) # [!code highlight]
    V = X @ W_v                         # (seq_len, d_v) # [!code highlight]

    d_k = Q.shape[-1]
    scores = (Q @ K.T) / np.sqrt(d_k)   # (seq_len, seq_len) # [!code highlight]

    if mask is not None:
        scores = np.where(mask == 1, scores, -1e9)

    weights = softmax(scores, axis=-1)  # attention distribution # [!code highlight]
    out = weights @ V                   # (seq_len, d_v) # [!code highlight]

    return out, weights
```

## Tiny worked example

Assume we have 4 tokens and model dimension 8.

```python
np.random.seed(42)

seq_len = 4
d_model = 8
d_k = d_v = 8

X = np.random.randn(seq_len, d_model)
W_q = np.random.randn(d_model, d_k)
W_k = np.random.randn(d_model, d_k)
W_v = np.random.randn(d_model, d_v)

out, weights = scaled_dot_product_attention(X, W_q, W_k, W_v)
# [!code highlight]

print("attention weights shape:", weights.shape)  # (4, 4)
print("output shape:", out.shape)                 # (4, 8)
print("row sums:", weights.sum(axis=-1))          # each row ~ 1.0
```

## PyTorch version (same math, tensor-first)

Here is the equivalent implementation in PyTorch:

```python
import torch
import torch.nn.functional as F

def scaled_dot_product_attention_torch(X, W_q, W_k, W_v, mask=None):
    """
    X:    (seq_len, d_model)
    W_q:  (d_model, d_k)
    W_k:  (d_model, d_k)
    W_v:  (d_model, d_v)
    mask: (seq_len, seq_len), optional. 1=keep, 0=block
    """
    Q = X @ W_q  # [!code highlight]
    K = X @ W_k  # [!code highlight]
    V = X @ W_v  # [!code highlight]

    d_k = Q.size(-1)
    scores = (Q @ K.transpose(-2, -1)) / (d_k ** 0.5)  # [!code highlight]

    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))  # [!code highlight]

    weights = F.softmax(scores, dim=-1)  # [!code highlight]
    out = weights @ V  # [!code highlight]
    return out, weights
```

And a tiny run:

```python
torch.manual_seed(42)

seq_len = 4
d_model = 8
d_k = d_v = 8

X = torch.randn(seq_len, d_model)
W_q = torch.randn(d_model, d_k)
W_k = torch.randn(d_model, d_k)
W_v = torch.randn(d_model, d_v)

mask = torch.tril(torch.ones(seq_len, seq_len))
out_t, weights_t = scaled_dot_product_attention_torch(X, W_q, W_k, W_v, mask=mask)
# [!code highlight]

print(weights_t.shape)  # torch.Size([4, 4])
print(out_t.shape)      # torch.Size([4, 8])
```

In production, you’ll usually rely on `torch.nn.MultiheadAttention` or `torch.nn.functional.scaled_dot_product_attention`, but writing it manually once makes the mechanism click.

What to notice:

- `weights[i, j]` is how much token `i` attends to token `j`
- each row sums to `1` because of softmax
- `out[i]` is a weighted blend of all value vectors

## Causal masking (decoder-style attention)

In autoregressive generation, token _t_ must not see future tokens `t+1...`.
We use a lower-triangular mask:

```python
def causal_mask(seq_len):
    return np.tril(np.ones((seq_len, seq_len), dtype=np.int32))

mask = causal_mask(seq_len=4)
out_masked, weights_masked = scaled_dot_product_attention(X, W_q, W_k, W_v, mask=mask)
```

This enforces left-to-right information flow.

## Multi-head attention in plain words

A single attention head learns one notion of relevance. Multi-head attention runs several attention heads in parallel so each head can specialize (e.g., syntax, coreference, positional patterns).

High-level steps:

1. Project `X` into multiple sets of `Q, K, V`
2. Run scaled dot-product attention per head
3. Concatenate head outputs
4. Apply final linear projection

## Practical intuition

- If two words are semantically related, their query-key score tends to be higher.
- Scaling by `sqrt(d_k)` prevents dot products from growing too large.
- Softmax turns raw scores into a probability-like distribution.

That’s the full mechanism behind the magic phrase “the model attends to relevant context.”

## Final takeaway

Self-attention is just:

1. score token relationships,
2. normalize scores into weights,
3. blend value vectors using those weights.

Once this clicks, the rest of the Transformer stack (multi-head attention, residuals, layer norm, feed-forward blocks) becomes much easier to understand.
